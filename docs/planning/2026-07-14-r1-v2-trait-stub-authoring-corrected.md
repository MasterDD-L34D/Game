## 0. Why v1 failed (5 misses to NOT repeat)

A single execution session hit five recon/generated-vs-source misses. Each is a
concrete trap for the next author:

1. **Derived layers**: a trait count-change stales THREE CI-drift-guarded bundles
   (`data/traits/index.csv`, `data/derived/analysis/*`, `reports/qa_*`). The local
   validator trio passes while they are stale -> **LOCAL-GREEN != CI-GREEN**.
2. **index.json is the indice-unico SOURCE** for `traits.<id>` (tier/slot/sinergie),
   NOT regenerated by `build_trait_index.js` (that writes only the CSV). Field changes
   (sinergie) must be **hand-synced** into index.json or `trait_template_validator`
   coverage / Codex P1 fires.
3. **Wrong i18n file**: `locales/it/traits.json` is a legacy/trait-editor dataset that
   is a **SYNC TARGET** (propagated FROM glossary by `scripts/sync_trait_locales.py`),
   NOT the game i18n. The authoritative runtime i18n is `data/i18n/it/traits.json`
   (imported by `apps/play/src/i18n.js`, curated by PR #3247).
4. **Shadow-duplicate**: 32 of the 46 stubs ALREADY have curated `data/i18n` entries
   (#3247). Run an "does it already exist?" check before authoring any prose.
5. **Generated catalogs**: `packs/evo_tactics_pack/docs/catalog/env_traits.json` and
   siblings are generated by `npm run sync:evo-pack`; hand-editing them breaks the
   `dataset-checks` no-drift guard. Regenerate, never hand-edit.

## 1. The real trait-authoring workflow (ground-truthed 2026-07-14)

Layers and their SoT (authoritative source):

| Data                  | File                                                                  | SoT?                             | How to change                                                                             |
| --------------------- | --------------------------------------------------------------------- | -------------------------------- | ----------------------------------------------------------------------------------------- |
| Design fields         | `data/traits/<fam>/<id>.json` (`sinergie`, `conflitti`, `slot`)       | YES (file-first, ADR-2026-06-19) | hand-edit                                                                                 |
| Consolidated index    | `data/traits/index.json` (`traits.<id>`)                              | hand-maintained aggregate        | hand-sync to match per-file                                                               |
| label/description     | `data/core/traits/glossary.json` (`label_it/en`, `description_it/en`) | YES                              | hand-edit, <=160 chars                                                                    |
| Legacy locale         | `locales/it/traits.json`                                              | NO (derived)                     | `python scripts/sync_trait_locales.py`                                                    |
| **Game runtime i18n** | `data/i18n/it/traits.json` (`traits.<id>`)                            | YES for player display           | hand-edit; schema `{label, mutazione_indotta, spinta_selettiva, uso_funzione, debolezza}` |
| CSV mirror            | `data/traits/index.csv`                                               | derived                          | `node scripts/build_trait_index.js`                                                       |
| Derived analysis      | `data/derived/analysis/*`                                             | derived                          | `python scripts/generate_derived_analysis.py --update-readme`                             |
| QA baselines          | `reports/qa_*`                                                        | derived                          | `npm run export:qa`                                                                       |
| Evo-pack catalog      | `packs/.../catalog/env_traits.json` + published copies                | generated                        | `npm run sync:evo-pack`                                                                   |

Authoring guide: `apps/trait-editor/docs/howto-author-trait.md`. Curator-gated
(Trait / Species / Biome Curator). **`uso_funzione` in data/i18n carries the MECHANIC
line** (matches `active_effects.yaml`), it is NOT pure flavor -> authoring it for a
combat trait requires grounding on the trait's active_effects entry (this overlaps R2).
