C · Skip rules

Condizione Azione
Setup > 12 min (ngrok rotto) LAN-only Vite IP locale via WhatsApp. Continua.
T01 > 14 min Skip T02 → vai T03 (hazard più interessante).
Party wipe < 5 turni Non riprovare. Nota + continua.
T03 hazard confonde (3+ domande verbali) Stop. Spiega 30s. Restart same. 1 retry max.
T04 > turno 15 senza win Stop. Vai T05 o HC.
T05 inizia dopo 1:15 Skip T05 → Debrief. HC richiede comfort party.
HC inizia dopo 1:20 NON iniziare HC. Vai diretto Debrief.
Backend crash mid-session Host npm run demo restart. Reconnect via stesso ngrok URL. Sessione in-memory persa = nuova.
WS reconnect loop > 30s Fallback LOBBY_WS_ENABLED=false. Host pilot, phone spectate.

D · Materials needed

Hardware

  • 1 laptop host — Chrome/Firefox + monitor o HDMI TV
  • 1 TV o 42"+ monitor + cavo HDMI
  • 4 phone (Android/iOS Chrome) o 4 browser su 4 device
  • Wi-Fi router — same network (LAN fallback)

Software / accounts

  • ngrok account (free) — pre-auth ngrok config add-authtoken <token>
  • Node 22 + npm su host
  • Repo npm ci done · npm run play:build done T-30

Session materials

  • 1 copia stampata questo playbook A4 fronte ×2
  • 1 penna per observer + notepad friction notes
  • Timer fisico (phone timer 90 min)
  • Bug report tab aperta su 2nd browser

Backup

  • WhatsApp/Signal group ngrok URL pronto
  • Comando memorizzato LOBBY_WS_ENABLED=false npm run demo

E · Pre-session checklist T-30 min

  • git pull origin main + npm ci + npm run play:build verde
  • node --test tests/ai/*.test.js → 311+/311+ pass
  • npm run demo in 1 terminal — preflight 5/5 verde
  • ngrok tunnel up + URL copiato clipboard
  • Browser host punta URL ngrok + lobby render OK
  • Phone test (anche solo browser incognito) join lobby + roster aggiornato
  • HDMI TV input · canvas full-screen prova responsive
  • Stampa cartacea presente · penna · timer
  • Tab "bug report" aperta + URL copiato
