# Legacy Index Mapping

Entry-point canonico unico:

- `docs/hubs/README.md`

Entrypoint legacy mantenuti attivi in dual-track:

- `docs/README.md` -> canonical hub
- `docs/INDEX.md` -> canonical hub
- `docs/00-INDEX.md` -> canonical hub

Regola operativa:

- nuovi riferimenti cross-team devono puntare al canonical hub
- gli index legacy restano consultabili ma non normativi
- stato/owner dei legacy index e' governato dal registry centrale
