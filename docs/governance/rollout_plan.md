# Governance Rollout Plan (6-8 weeks)

## Week 1-2

- Schema metadata e registry centrale attivi.
- Hub canonici per workstream pubblicati.
- Workflow CI docs-governance in warning-only.

## Week 3-4

- Tag massivo documenti critici (`doc_status`, owner, review cycle).
- Riduzione mismatch tra frontmatter e registry.
- Prima baseline di report drift settimanali.

## Week 5-6

- Chiusura gap moduli incomplete prioritari.
- Consolidamento indici legacy verso entrypoint canonico.
- Definizione elenco check da promuovere a required.

## Week 7-8

- Promozione graduale dei gate docs-governance a required.
- Aggiornamento branch protection.
- Retro finale con metriche: stale docs, mismatch, coverage metadata.
