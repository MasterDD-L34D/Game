# Session audit -- 2026-06-18 build-residui (post-merge adversarial review)

> Method: Workflow 7 adversarial reviewers (1/PR, refute-by-default, ground-truth git show) + synth-critic (verify/dedup/cross-cutting/fix-plan). 8 agents, 19 raw findings -> 18 verified (1 refuted). Codex did NOT review the 6 Game PRs (rate-limit); GGv2 #481 yes (2 P2 fixed pre-merge). All confirmed findings have since been FIXED -- see Resolution at the bottom.

## Verdict

The 2026-06-18 session work is largely sound and SPEC-faithful. I adversarially re-read every cited file against origin/main (3dbbec68) for Game and 200ac70 for Game-Godot-v2, and ground-truth-ran the codex validator and the HA5 proxy against the real corpus. Mechanics are correct: evalNpcGates reproduces the canonical recruit/mating constants exactly; the codex loader/route are additive and read-only; the HA5 proxy never serializes the raw float; the frontend XSS surface is fully escaped; the Godot recruit path's two Codex P2 fixes are genuinely present. Of 18 reviewer findings I confirm 14, partially confirm 1, refute 1 (the '9/9 vs 8 tests' provenance slip is FALSE -- origin/main has 9 test() calls, matching the commit), and the remaining 2 are accurate-but-benign latent gaps. Only ONE finding is genuinely merge-blocking-class: the HA2 codex authoring-gate validator (#2833) does NOT run on codex-only PRs because data/codex/** is absent from the ci.yml 'stack' paths-filter -- anti-pattern #10 is NOT actually closed for the exact change class the gate exists to police (ground-truth: codex-only edit triggers data==true -> dataset-checks, which never runs npm run test:api / tests/codex). Two cross-cutting design issues dominate the rest: (1) the SECRET coherence-score invariant (#2828/#2835) holds only by author data-discipline -- the loader serves the raw YAML whole with no allowlist/schema, so a future entry could leak a novel-named score field; (2) the HA5 'endemica/inattesa' descriptor reads as an ECOLOGICAL-FIT claim but the proxy measures YAML link/completeness -- I proved live the same content-rich dune_stalker flips endemica(1.0)->adattata(0.4)->inattesa(0.1) purely by deleting cross_ref/habitat/hooks with zero ecological-prose change. Net: ship-safe as merged (the gate gap and semantics drift are forward-work, not active regressions), but the CI blind-spot should be fixed before HA2 is trusted as a merge gate, and the descriptor semantics need an explicit acceptance-trail note.

## Cross-cutting

- SECRET coherence-score invariant (sez.8) spans #2828 (loader serves raw YAML whole) and #2835 (HA5 proxy correctly never serializes the raw float). Verified the invariant HOLDS today: the scorer subsystem never writes data/codex, and the one example file carries no score field. But it is enforced by author data-discipline, not code -- no allowlist, no AJV schema, and the HA2 validator that could catch a stray field does not even run on codex-only PRs (the P1). A SINGLE fail-closed serialization allowlist in summarize()+getCodexEntry() hardens #2828 and #2835 together; pair it with the P1 CI wiring so the gate actually fires.
- Codex-entry-id == unit species_id is an undocumented contract that is currently violated in practice. #2830 keys unlock on the archetype label (species) while codex ids are canonical species slugs surfaced only via encounter species_hint (#2828/example). The same id-namespace question will recur for any future surface that joins a runtime unit to a codex/species entry (e.g. the deferred mating_success trigger, a 'Specie' detail deep-link from combat). Fix the propagation once (species_hint -> unit.species_id) and document the equality contract, rather than patching each consumer.
- HA2 authoring-rubric semantics drift across two layers: (a) the validator's SOFT band (100-300) and (b) the HA5 proxy both encode 'what good authoring looks like', yet both diverge from the runtime alienaCoherence scorer (different field names: codex-local story_hook_it/sistema_relation vs runtime narrative_hooks/lore_ref/narrative_tag; completeness-proxy vs ecological-fit). This is a deliberate codex-native re-scope (documented + master-dd-ratified), but the three coherence surfaces (validator / HA5 descriptor / runtime scorer) now measure three different things under one A.L.I.E.N.A. banner. Worth one acceptance-trail note reconciling the three lenses so a future reader does not assume they agree.
- Recruit authority model is coherent across the seam: Game #2826 evalNpcGates reproduces the canonical constants (RECRUIT_AFFINITY_MIN=0/RECRUIT_TRUST_MIN=2/MATING_TRUST_MIN=3) and the GGv2 #481 presenter prefers the server-enriched can_recruit/can_mate with a has()-guarded client fallback using the identical constants -- no threshold drift. The Nido recruit path correctly carries affinity_at_recruit=0 (no-bypass) vs debrief=1. The contract between npg #2826 and the Godot presenter #481 is sound; the only #481 gap is the freed-view lifecycle guard, orthogonal to the authority model.
- Frontend resilience pattern is inconsistent within #2829: the list view degrades gracefully-ish (but with the bare '0' message) while the detail view has a hard dead-end (no back button + leaked _specieSelectedId). Both stem from error/catch branches not restoring navigable state. A shared 'always render an escape affordance + reset selection state on error' convention would close the P2 dead-end, the P3 '0' message, and prevent the same class in future Codex sub-views.
